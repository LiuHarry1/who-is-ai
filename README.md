# Who is AI？| 人类鉴别测试游戏

多人聊天室版"图灵测试"游戏：20 名玩家分成技术房和生活房，通过两轮聊天与投票找出隐藏在其中的 AI。第一轮投票结果会实时改变 AI 的第二轮行为（高怀疑 → 简短被动，低怀疑 → 主动细节）。

## 架构

- `server/`：Node.js + TypeScript + Socket.IO。游戏状态机（服务端权威）、AI 玩家引擎（人格卡 + 拟人化调度）、JSON 快照落盘。
- `web/`：React + Vite + Tailwind。玩家端（聊天/投票）、主持人控制台、揭晓大屏。
- AI 通过 OpenAI 兼容接口调用（`claude-ops-4.6`，失败降级 `gpt-4.1`，再失败走预设 fallback 回复库）。

## 快速开始（本地开发）

```bash
# 终端1：后端（默认 :3000）
cd server && npm install && npm run dev

# 终端2：前端（默认 :5173，已配置 socket 代理到 :3000）
cd web && npm install && npm run dev
```

确保 OpenAI 兼容代理运行在 `http://localhost:4141`（可通过 `.env` 修改，参考 `.env.example`）。

## Docker 部署

```bash
docker compose up -d --build
```

- 服务跑在 `:3000`，前端静态文件由同一进程托管。
- LLM 代理在宿主机时，默认的 `http://host.docker.internal:4141/v1` 即可工作（Linux 已配 `host-gateway`）。
- 游戏状态快照持久化在 `./data/`，容器重启不丢。

## 入口

| 页面 | 地址 | 说明 |
|---|---|---|
| 玩家入口 | `/#/` | 选择技术房 / 生活房，输入真名（仅主持人可见），系统分配匿名代号 |
| 技术房直达 | `/#/room/tech` | 可做成二维码 |
| 生活房直达 | `/#/room/life` | 可做成二维码 |
| 主持人控制台 | `/#/host` | 口令默认 `whoisai`（env `HOST_KEY`） |

## 主持流程

1. **等待大厅**：玩家扫码进入；默认 AI 配置为技术房 1 个、生活房 2 个（隐藏第二个），LOBBY 阶段可加减 AI。
2. **规则讲解**：点"下一阶段"，系统洗牌分配代号（玩家01…）；AI 同时预生成第一轮答案。
3. **两轮聊天/投票**：聊天 5 分钟、投票 3 分钟自动流转；可随时"加时 1 分钟"或手动跳转。
4. **揭晓**：进入"揭晓大屏"——两轮投票对比 → 按票数从低到高逐个揭晓 → 颁奖（最强侦探 / 最强演员 / 最强 AI）。

## 服务端强制规则

- 每轮每人最多 5 条发言，发言间隔至少 2 秒
- 第二轮每人最多 @ 一人
- 投票不能投自己，理由必填
- 投票阶段聊天输入锁定
