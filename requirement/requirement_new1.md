# 《Who is AI?》试玩反馈 → 变更需求（相对当前实现）

基于 `requirement_new.md` + 当前代码（`server/` / `web/`）整理。  
本文只写**相对现状要改什么**；未列出的规则沿用 `requirement_new.md`。

---

## 最高宗旨（贯穿全程，改 prompt / 调度时必须遵守）

> **你的唯一目标：不让人识别出你是 AI。**

游戏过程中，所有 AI 相关提示词与行为策略都必须服从这一点：

- **说话要像真人**：口语、短句、随手打字感；跟着群里已有聊天的语调、节奏、话题走，而不是「答题」或「客服回复」。
- **回不回、说什么、说多长，都以「别露馅」为准**——不是礼貌、不是完整、不是围着主问题。
- 下文 §1 / §2 / §8 都是这条宗旨的具体落地；若某条细则与本宗旨冲突，**以本宗旨为准**。

**实现落点：** `aiConfig.ts` → `baseRules` 开篇写死此目标；`prompts.ts` 各条 user prompt 不写与之冲突的硬性指令（如「必须正面回应 @」）。

## 1. AI 发言节奏与长度（提示词 + 调度器）

### 1.1 等「正式聊天」开始再开口

**问题（试玩记录）：** 人类还在刷短消息 / 代号 / 互指 AI 时，AI 已抛出完整「踩坑小作文」。

**期望：**

- 进入 `ROUND1_CHAT` / `ROUND2_CHAT` 后，**不要立刻**发主问题答案。
- 至少等到本轮出现 **≥2 条非系统、非 AI 的人类消息** 后，AI 才允许首次发言（含主答与接话）。
- 若超时仍无人说话（建议 **90s**），才允许 AI 主动开口，避免冷场。

**实现落点：** `server/src/ai/scheduler.ts`（现为固定 `rand(15000, 70000)` 后 `postMainAnswer`）。

### 1.2 单条消息尽量 ≤ 12 个字（仅提示词，不做硬截断）

**期望：**

- **只改提示词**：在 `baseRules` / 主答与接话的 user prompt 里写明「一般不超过 12 个字，像微信群随手打的短句」。
- **不做** `sanitizeOutput` 硬截断到 12 字（保留现有净化逻辑即可，不必为此改上限）。
- 宁可短答，不要完整故事；偶发略超可接受，靠模型自律，不靠代码砍句。

**实现落点：** `aiConfig.ts` → `baseRules`；`prompts.ts` → `buildMainAnswerPrompt` / `buildReplyPrompt` 字数说明。

### 1.3 不必围着主问题答；语调跟已有聊天走

**期望：**

- 取消「开局必发一条主问题答案」的强流程，或改为：群里已经在聊别的时，**可以完全不答主问题**，只顺着最近几条消息说话。
- **所有回复的语调、长短、话题，都应贴合当前聊天记录**（群里在刷梗就短跟；在互指 AI 就别突然讲机场踩坑；有人吐槽就接吐槽）。
- 允许跑题、吐槽、互怼、短附和；不要每次都回到「踩坑饭 / 旅行目的地」。

**实现落点：** `scheduler.postMainAnswer` 触发条件；`buildReplyPrompt` / `buildMainAnswerPrompt`；`baseRules`。

### 1.4 参考语气（试玩摘录，改 prompt 用）

真人侧常见：极短（「没有」「w」）、互相点名是不是 AI、跑题、脏话玩笑、不接话。  
AI 侧问题：话密、字数长、句式完整、被点名必回、总把话题拉回「机场/外卖踩坑」。

完整记录见文末「附录」。

## 2. 群里没聊主题时，AI 也不要聊主题

**期望：**

- 若最近若干条人类消息**明显偏离**房间主题（互指 AI、刷梗、人身攻击玩笑、无关闲聊等），AI **禁止**主动抛美食/旅游经历。
- 此时只允许：极短附和、吐槽气氛、或沉默（见 §8）。
- 房间上下文（`roomContextFood` / `roomContextTravel`）改为「主题是可选背景，不是每条必聊」；领域素材（domain notes）在跑题阶段不要强行注入到回复目标里。

**实现落点：** `aiConfig.prompts`；`buildReplyPrompt` 的 trigger 说明；可选：调度层根据最近消息做「主题相关度」粗判后再决定是否主动发言。

## 4. 主持人投票展示：揭晓前隐藏身份

**背景：** 主持人页会投到大屏，全场可见。

**期望（游戏未到揭晓前，含 `ROUND1_VOTE` / `ROUND2_VOTE` 及之前）：**

- 投票列表只显示：**代号 → 代号 + 理由**。
- **禁止显示：** 投票人/被投人的真实姓名、是否为 AI。
- 真实姓名与 AI 身份**仅在主持人执行揭晓（大屏揭晓流程）之后**才可出现。

**现状问题：**

- `Host.tsx` 投票 Tab：`voter?.isAI && (AI)`。
- 聊天气泡：`AI · {realName}` / `{realName}`。
- 玩家列表 Tab：直接展示真名与 AI 标记（若投屏也会泄密）。

**范围（已拍板 · 方案 A）：**

- 揭晓前，Host 的「投票」「聊天」「玩家」列表都只显示代号。
- 真名 / 是否 AI 仅在揭晓流程之后可见。

## 5. 游戏开始后不能加人

**期望：**

- 允许加入的阶段：**仅 `LOBBY`**。
- 从主持人进入 `RULES`（或等价「游戏开始」）起，拒绝新人类 join，提示如「游戏已开始，无法加入」。
- 已入局玩家凭 token 重连仍允许（与现有 reconnect 一致）。

**现状：** `engine.join` 只在 `phase === 'REVEAL'` 时拒绝。

**实现落点：** `server/src/game/engine.ts` → `join`。

## 6. 主持人聊天界面：游戏中实时展示（投屏用）

**期望（修订：投屏需要大家看到聊天）：**

- 进入聊天阶段后，Host 控制台**保留并展示「聊天」Tab**，实时滚动本局消息。
- 揭晓前聊天气泡**只显示代号**，不显示真名 / 是否 AI（与 §4 方案 A 一致）。
- 进入 `ROUND1_CHAT` / `ROUND2_CHAT` 时自动切到聊天 Tab，方便投屏。

**实现落点：** `web/src/pages/Host.tsx` → `RoomPanel`。

## 6.1 多个 AI 不得连麦

**期望：**

- 若房间有 ≥2 个 AI：任一 AI 发言后，**至少再有 1 条人类发言**，另一个（或同一个）AI 才允许再开口。
- 禁止 AI→AI 连发；允许 AI→人→AI。

**实现落点：** `server/src/ai/scheduler.ts` → `RoomSpeakGate.humanGapReady`。

## 7. 去掉 LLM 复盘

**期望：**

- 不再在进入 `REVEAL` 时调用 LLM 生成复盘。
- 大屏流程去掉「LLM 复盘」步骤（现：① 投票对比 → ② 逐个揭晓 → ③ LLM 复盘 → ④ 颁奖 → 改为 ①② + 颁奖）。
- 相关代码可停用或删除：`server/src/ai/recap.ts`、`index.ts` 中 REVEAL 触发、`Host.tsx` 的 `RecapView` / step。

**说明：** 阵营胜负判定、揭晓、颁奖保留。

## 8. 被 @ 可不回；宗旨是别被认出是 AI

**期望：**

- 取消「被 @ 必须回复」；被 @ 了也可以当没看见（真人常见）。
- **最高宗旨：所有发言/沉默策略，都服务于「不要让人识别出你是 AI」。** 回不回、回什么、多长，都以混进真人里为准，而不是礼貌接话或完整答题。
- 若选择回复：内容与**语调必须结合已有聊天记录**（跟群里的节奏、用词、玩笑程度），不要跳出氛围单独「认真回答」。
- Prompt 去掉「必须正面回应 @」类硬性指令（`buildReplyPrompt` 现状写死 must 回应）；改为「可回可不回；回则短、贴聊天记录」。
- 调度层：去掉 `mustReply=true` 强制路径；被 @ 时用概率决定是否回（不必固定 50%，以「像真人、别露馅」为准即可）。

**实现落点：** `scheduler.onMessage`（`mustReply`）；`prompts.buildReplyPrompt`；`baseRules`。

## 已拍板

- **最高宗旨：** 提示词必须像人说话；目标是**不让人识别出你是 AI**（见文首）。
- **12 字：** 只写入提示词约束，**不做硬截断**。
- **被 @：** 可以不回；一切以「别被认出是 AI」为宗旨；回复语调结合已有聊天内容。
- **正式聊天门槛：** 本轮 **≥2 条**非系统、非 AI 的人类消息后，AI 才允许首次发言；超时 90s 无人说话才可主动开口。
- **Host 身份隐藏：** **方案 A** — 揭晓前，投票 / 聊天 / 玩家列表都只显示代号；真名与是否 AI 仅揭晓后可见。
- **Host 聊天投屏：** 游戏中**显示**实时聊天（仅代号）；进入聊天阶段自动切到聊天 Tab。
- **AI 间隔：** 两条 AI 发言之间至少隔 **1 条人类消息**。
- **聊天语料：** 两轮聊完后可复制 / 保存为独立文件；仅人类发言入库；下一局 AI 抽样参考语气（见 §9）。

---

## 9. 真人聊天语料库（已实现）

**目标：** 把真实对局里的人类说话方式积累下来，供后续局 AI 参考，更容易混进真人。

**流程：**

1. 进入 `ROUND2_VOTE` / `REVEAL` 后，主持人控制台与揭晓大屏出现语料操作区。
2. **复制人类发言** / **复制完整记录**：写入剪贴板，方便外发或人工挑拣。
3. **保存为语料**：服务端写入 `data/chat-corpus/<food|travel>/<时间戳>-xxx.md`，**一局一个文件**；文件内**只含人类发言**（去掉系统消息与 AI）。
4. 之后 AI 生成回复时，从该房间语料中**抽样短句**注入 system prompt（学语气/长短，禁止照抄经历）。
5. 主持人「设置」页可查看语料列表并删除质量差的文件。

**实现落点：** `server/src/ai/chatCorpus.ts`；`prompts.ts` 注入；`Host.tsx` / `AdminPanel.tsx`；`host:action` 的 `saveCorpus` / `getTranscript`。
